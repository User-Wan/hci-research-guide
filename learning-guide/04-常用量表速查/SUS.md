# SUS 系统可用性量表

## 概述

**SUS**（System Usability Scale）是一个 10 项问卷，用于测量感知可用性。它由 John Brooke 于 1986 年开发，是 HCI 研究中最常用的可用性量表之一。

---

## 量表内容

SUS 包含 **10 个问题**，交替使用正向和反向措辞：

| 序号 | 问题 | 类型 |
|------|------|------|
| 1 | 我认为我会经常使用这个系统 | 正向 |
| 2 | 我认为这个系统不必要地复杂 | 反向 |
| 3 | 我认为这个系统很容易使用 | 正向 |
| 4 | 我认为我需要技术人员的帮助才能使用这个系统 | 反向 |
| 5 | 我发现这个系统的各种功能很好地整合在一起 | 正向 |
| 6 | 我认为这个系统存在太多不一致性 | 反向 |
| 7 | 我认为大多数人会很快学会使用这个系统 | 正向 |
| 8 | 我发现这个系统使用起来非常麻烦 | 反向 |
| 9 | 我使用这个系统时感到非常自信 | 正向 |
| 10 | 在使用这个系统之前，我需要学习很多东西 | 反向 |

---

## 使用方法

### 步骤 1：任务完成后填写

参与者在完成任务或使用系统后填写问卷。

### 步骤 2：评分说明

```
每个问题都有一个从 1 到 5 的李克特量表：

非常不同意 | 不同意 | 中立 | 同意 | 非常同意
    1        2       3      4        5
```

### 步骤 3：计算 SUS 分数

**计分方法**：

1. **奇数题**（正向）：得分 - 1
2. **偶数题**（反向）：5 - 得分
3. 将所有调整后的分数相加
4. 乘以 2.5 得到最终 SUS 分数

**公式**：
```
SUS 分数 = [(Q1-1) + (5-Q2) + (Q3-1) + (5-Q4) + (Q5-1) + (5-Q6) + (Q7-1) + (5-Q8) + (Q9-1) + (5-Q10)] × 2.5
```

**分数范围**：0-100

## 分数解释

### 等级划分

| 分数范围 | 等级 | 百分位 | 说明 |
|----------|------|--------|------|
| > 80 | A | 90-99% | 优秀 |
| 70-80 | B | 70-89% | 良好 |
| 50-70 | C | 30-69% | 一般 |
| < 50 | D | 0-29% | 较差 |

### 行业基准

| 分数 | 说明 |
|------|------|
| **68** | 平均可用性（50 百分位） |
| **50** | 可接受的最低分数 |
| **70+** | 良好的可用性 |
| **80+** | 优秀的可用性 |

### Adjective Ratings

| 分数范围 | 形容词 |
|----------|--------|
| > 80 | Best Imaginable |
| 70-80 | Excellent |
| 60-70 | Good |
| 50-60 | OK |
| 40-50 | Fair |
| 30-40 | Poor |
| < 30 | Worst Imaginable |

## 数据分析

### 描述统计

计算 SUS 分数的：
- 均值（Mean）
- 标准差（Standard Deviation）
- 最小值、最大值
- 中位数（Median）

### 比较分析

**比较两种设计**：
- 使用配对 t 检验（组内设计）
- 使用独立样本 t 检验（组间设计）

**比较多组**：
- 使用重复测量 ANOVA（组内设计）
- 使用单因素 ANOVA（组间设计）

### 效应量

计算 Cohen's d 或 η² 来衡量效应大小。

## 重点总结

- SUS 是 **10 项问卷**，测量感知可用性
- 分数范围 **0-100**，68 分是平均水平
- **奇数题**正向计分，**偶数题**反向计分
- 最终分数 = 调整后分数之和 × 2.5
- **70+ 分**表示良好可用性
- 是 HCI 研究中最常用的可用性量表
