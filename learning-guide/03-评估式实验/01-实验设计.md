# 实验设计（Experimental Design）

## 学习目标

- 理解实验研究的基本概念和类型
- 掌握组间设计、组内设计和混合设计的区别和适用场景
- 学会设计一个完整的 HCI 实验

---

## 核心概念

### 三种研究类型

| 类型 | 定义 | 特点 |
|------|------|------|
| **真实验（True Experiment）** | 多条件 + 随机分配 | 可建立因果关系 |
| **准实验（Quasi-Experiment）** | 多条件 + 无随机分配 | 因果关系受限 |
| **非实验（Nonexperiment）** | 单一观察组/条件 | 无法建立因果关系 |

### 三种设计方法

| 设计类型 | 定义 | 优点 | 缺点 |
|----------|------|------|------|
| **组间设计（Between-Group）** | 每个参与者只体验一个条件 | 无学习效应 | 需要更大样本量 |
| **组内设计（Within-Group）** | 每个参与者体验所有条件 | 样本量小、控制个体差异 | 有学习效应和疲劳效应 |
| **裂区设计（Split-Plot）** | 部分变量组间 + 部分变量组内 | 结合两者优势 | 设计复杂 |

### 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| **自变量** | Independent Variable | 研究者操纵的变量 |
| **因变量** | Dependent Variable | 研究者测量的变量 |
| **混淆变量** | Confounding Variable | 可能影响结果的无关变量 |
| **学习效应** | Learning Effect | 参与者因练习而提高表现 |
| **疲劳效应** | Fatigue Effect | 参与者因疲倦而表现下降 |
| **交互效应** | Interaction Effect | 一个变量的影响取决于另一个变量 |

## 方法步骤

### 步骤 1：识别研究假设

**核心问题**：
- 你的研究问题是什么？
- 你的假设是什么？

**示例**：
> 研究问题：新界面设计是否比旧设计更高效？
> 假设：新界面的平均任务完成时间显著短于旧界面。

### 步骤 2：确定自变量和因变量

**自变量**（你要操纵的）：
- 界面类型（新 vs 旧）
- 任务类型（简单 vs 复杂）

**因变量**（你要测量的）：
- 任务完成时间
- 错误率
- 用户满意度

### 步骤 3：选择设计方法

**决策树**：

```
是否需要比较多个条件？
├── 否 → 非实验（描述性研究）
└── 是 → 能否随机分配参与者？
    ├── 是 → 真实验
    │   └── 选择设计类型
    │       ├── 学习效应大 → 组间设计
    │       ├── 个体差异大 → 组内设计
    │       └── 两者都有 → 裂区设计
    └── 否 → 准实验
```

### 步骤 4：控制混淆变量

**常见混淆变量**：
- 参与者特征（年龄、经验、教育）
- 环境因素（噪音、光线、温度）
- 实验者行为（语气、肢体语言）

**控制方法**：
- 随机分配
- 匹配参与者
- 标准化程序
- 使用书面说明

### 步骤 5：设计实验流程

**标准流程**：

```
1. 问候参与者
2. 介绍研究目的
3. 获得知情同意
4. 分配实验条件
5. 训练任务
6. 正式实验
7. 填写问卷
8. 事后说明
9. 支付报酬
```

### 步骤 6：进行预实验

**预实验目的**：
- 测试实验程序
- 发现潜在问题
- 估计所需时间

**预实验参与者**：3-5 人，来自目标人群

## 实践案例

### 案例：比较两种菜单设计

**研究问题**：下拉菜单 vs 侧边栏菜单，哪个更高效？

**实验设计**：

| 项目 | 设计 |
|------|------|
| **自变量** | 菜单类型（下拉 vs 侧边栏） |
| **因变量** | 任务完成时间、错误率 |
| **设计类型** | 组间设计（避免学习效应） |
| **参与者** | 20 人（每组 10 人） |
| **任务** | 找到指定功能并点击 |

**结果分析**：
- 计算两组的平均完成时间
- 进行 t 检验比较差异
- 报告效应量（Cohen's d）

## 常见错误

### ❌ 错误 1：不进行预实验

**问题**：直接进行正式实验，可能遗漏程序问题

**正确做法**：始终进行预实验，即使时间紧张

### ❌ 错误 2：混淆变量未控制

**问题**：参与者经验差异影响结果

**正确做法**：预筛选参与者，或使用组内设计

### ❌ 错误 3：样本量不足

**问题**：无法检测到统计显著性

**正确做法**：使用功效分析（Power Analysis）计算所需样本量

### ❌ 错误 4：顺序效应未平衡

**问题**：组内设计中，条件顺序影响结果

**正确做法**：使用拉丁方设计平衡顺序

## 重点总结

- **真实验**需要随机分配，可建立因果关系
- **组间设计**避免学习效应，但需要更大样本量
- **组内设计**控制个体差异，但有学习效应
- **预实验**是发现程序问题的唯一机会
- **混淆变量**必须被识别和控制
- **拉丁方设计**用于平衡组内设计的顺序效应
