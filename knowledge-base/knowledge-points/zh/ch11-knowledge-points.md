# 第11章：分析定性数据 — 知识点

## 1. 核心概念定义

**定性数据分析 (Qualitative Data Analysis)** 是将文本、多媒体和其他人工制品中发现的非结构化数据转化为关于所考虑情境或问题重要方面的详细描述的过程。与处理数值测量的定量分析不同，定性分析涉及对模糊文本和多媒体内容的人工编码，使其更容易受到偏见解释或主观操纵的影响。

**内容分析 (Content Analysis)** 是一种系统、可复制的技术，基于明确的编码规则将许多文字压缩成较少的内容类别。更广泛的定义允许其他类型的信息，将其定义为“任何通过客观和系统地识别消息的指定特征来进行推断的技术。”内容分析适用于文本信息以及绘图、音乐和视频等多媒体材料。

**编码 (Coding)** 是为文本块或其他定性数据分配类别和描述符的过程。它涉及与数据互动、在数据之间进行比较、派生概念来代表这些数据，然后根据这些概念的属性和维度来发展它们。编码远不止是解释或计算关键词。

**扎根理论 (Grounded Theory)** 是由 Glaser 和 Strauss 首次提出的定性研究方法，旨在发展“基于系统收集和分析的数据”的理论。它是一种归纳研究方法，从经验观察或数据开始，旨在发展有充分依据的理论，与从预先形成的假设开始的实验研究形成对比。

**先验编码 (A Priori Coding)** 涉及使用既定的理论或假设来指导编码类别的选择。类别来自先前发表的工作或研究人员自己先前的调查。这种方法使用起来更简单，但可能牺牲更广泛的洞察。

**涌现编码 (Emergent Coding)** 指在没有任何理论或模型指导分析的情况下进行的定性分析。研究人员从记录有趣的概念或想法开始，并不断提炼这些想法，直到出现一个连贯的模型。这是扎根理论的基础。

**实境编码 (In Vivo Code)** 是通过直接借用参与者自己描述实例或关键元素时使用的生动或准确的术语来生成的编码类别。这种方法有助于确保概念尽可能接近参与者自己的语言。

**研究者指定概念 (Researcher-Denoted Concepts)** 是当原始文本不包含描述感兴趣实例的关键术语时，由研究人员创建的编码术语。这些与直接来自参与者的实境编码形成对比。

**开放编码 (Open Coding)** 是扎根理论的第一阶段，分析文本以识别任何有趣的现象。每个独特的现象都被赋予一个独特的名称或代码。研究人员对数据中所有可能性持开放态度，不受预先建立的理论的约束。

**轴心编码 (Axial Coding)** 是识别和定义通过开放编码确定的概念和类别之间关系的过程。这是将代码分组为更高层次概念和类别的关键步骤。

**选择编码 (Select Coding)** 发生在从新兴理论的背景下重新审视先前编码的数据时。这允许理论的细化以及开放编码和轴心编码的进一步迭代。

**编码簿（命名法）(Codebook / Nomenclature)** 是一个编号类别的列表，旨在代表对特定问题的全部可能响应。它被组织成一个包含多个层次细节的层次结构。

**三角验证 (Triangulation)** 在定性分析中指使用多个数据源来支持解释。数据源可以是相同类型数据的不同实例（多个参与者）或完全不同的来源（观察和时间日记）。

**效度 (Validity)** 在定性研究中意味着使用完善且记录良好的程序来提高发现的准确性。它检查工具测量其预期测量内容的程度。

**信度 (Reliability)** 指结果的一致性：如果不同的研究人员处理共同的数据集并得出相似的结论，则这些结论被认为是可靠的。

**科恩卡帕系数 (Cohen's Kappa)** 是一种统计量，以 0 到 1 的等级评定评分者间信度，考虑了编码者之间偶然的一致性。值为 0 表示一致性完全是偶然的；1 表示完全可靠。值高于 0.60 表示令人满意的信度。

**编码者间信度 (Intercoder Reliability)** 检查不同的编码者是否以一致的方式编码相同的数据。它通过计算多个独立编码者之间的一致性来衡量。

**编码者内信度 (Intracoder Reliability)**（稳定性）检查同一编码者在整个编码过程中是否一致地评估数据。如果要求编码者多次编码相同的数据，编码应该是一致的。

## 2. 理论框架

**归纳与演绎分析范式 (Inductive vs. Deductive Analysis Paradigm)**：定性数据分析，特别是扎根理论，以归纳方式运作——从数据开始发展理论。这与从假设开始并演绎测试它们的实验研究形成对比。归纳方法允许理论通过多轮收集和分析从数据中涌现，产生一些人所谓的“逆向工程假设”。

**三阶段定性分析模型（科宾和斯特劳斯）(Three-Stage Qualitative Analysis Model - Corbin and Strauss)**：阶段 1 从数据集中识别主要主题和想法。阶段 2 深入每个组成部分以找到描述性属性和维度，并理解它们的相互关系。阶段 3 利用研究单个组成部分的知识对原始内容进行推断。该模型强调在阶段之间的迭代移动。

**内容分析框架 (Content Analysis Framework)**：内容分析对两类内容进行操作：媒体内容（出版物、广播、网站、录音）和受众内容（通过调查、访谈、焦点小组、日记、观察收集的反馈）。基于文本和基于多媒体的信息都使用具有明确规则的系统编码程序进行分析。

**编码方法连续体 (Coding Approaches Continuum)**：从纯粹的涌现编码（扎根理论）到纯粹的先验编码（理论框架）的光谱。许多研究使用两者的混合。涌现编码提供更广泛的洞察但更复杂；先验编码更简单但可能错过意外的模式。选择取决于研究领域的成熟度和既定理论的可用性。

**效度框架 (Validity Framework)**：效度的三种主要方法——表面效度（主观内容评估）、效标效度（预测经过验证的测量）和构念效度（通过因子分析测试理论构念）。在 HCI 中，效度通过记录良好的数据库、证据链、数据源三角验证和考虑替代解释来建立。

**信度框架 (Reliability Framework)**：信度跨越两个维度——编码者内信度（稳定性：同一编码者，随时间一致）和编码者间信度（可复制性：不同编码者，一致编码）。测量指标包括百分比一致性（简单但不考虑偶然性）和科恩卡帕系数（考虑偶然一致性）。最终目标是确保不同的人以相同的方式编码相同的文本。

## 3. 方法论步骤

### 步骤1：准备分析

在开始分析之前，明确定义数据集并删除任何不符合标准的数据。仔细研究数据以了解其来源的人群。理解数据的具体上下文——在脱离上下文的情况下分析数据是没有意义且高度偏见的。在定义人群时考虑年龄、性别、职业、教育和领域经验等因素。

### 步骤2：初步阅读和沉浸

在开始任何编码之前，从头到尾阅读文本。在第一轮中，抵制写注释或代码的冲动。目的是沉浸在参与者的生活和体验中，并在关注任何特定方面之前获得数据集的总体、无偏见的想法。

### 步骤3：选择编码方法

在涌现编码（扎根理论）、先验编码（理论框架）或两者混合之间选择：
- 对于文献有限的新主题：使用基于扎根理论的涌现编码
- 对于有既定理论的主题：使用先验编码指导类别选择
- 对于复杂的研究：根据需要结合两种方法

### 步骤4：进行开放编码

分析文本以识别有趣的现象。寻找特定类型的陈述：目标、行动、结果、后果、原因、背景和策略。对于每个现象，要么使用参与者自己的话（实境编码），要么创建研究者指定概念。在编码期间：
- 寻找可能携带有价值信息的关键项目
- 不断对数据提出问题（敏感化和理论问题）
- 在各个层面进行比较（类别之间、组之间、与文献之间）

### 步骤5：发展概念和类别

将描述相似内容的代码集合分组形成更高层次的“概念”，然后将概念分组为“类别”。构建概念和类别的定义。通过轴心编码识别和定义概念之间的关系。构建具有多层细节的层次编码簿结构。

### 步骤6：构建理论

创建关于数据中现象的推断性和预测性陈述。在概念和类别之间发展明确的因果联系或相关性。通过从新兴理论的背景下重新审视先前编码的数据来进行选择编码。在数据收集和分析之间进行迭代以完善理论。

### 步骤7：确保效度

建立一个包含所有收集材料和分析产品的数据库。建立将分析结果链接回原始数据的证据链。使用数据源三角验证，多个独立数据指向共同方向。开发和考虑替代解释。给予每个数据点适当的关注和审查。

### 步骤8：确保信度

在开始时制定明确的编码说明。培训所有编码者充分理解说明。测试编码一些数据并计算信度测量。如果未达到期望的信度，重新培训、讨论分歧并修订编码簿。在整个正式编码过程中进行频繁的信度检查。计算科恩卡帕系数——值高于 0.60 表示令人满意的信度。

### 步骤9：分析多媒体内容

应用与文本内容相同的指南。识别要描述或注释的关键实例。考虑使用部分自动化方法：手动注释子集，用它训练自动化系统，然后自动注释其余部分。评估注释的信度。对于大型数据集，由于人工成本高，选择子集进行手动分析。

## 4. 决策标准

**何时使用定性分析**：当研究涉及文本、观察、视频和人工制品以理解复杂情境时使用。适用于民族志、案例研究、访谈、焦点小组和调查的数据，这些数据无法进行数值测量或数值测量不是目标。

**涌现编码与先验编码**：当研究文献有限且没有既定理论的新主题时，选择涌现编码。当存在可以指导类别选择的既定理论或框架时，选择先验编码。当某些方面有理论基础而其他方面是探索性时，使用混合方法。

**选择编码者**：主观编码者（研究设计者）具有领域专业知识，需要最少的培训，但可能具有夸大的信度和受限的思维。客观编码者对新模式更开放，偏见更少，但需要大量培训，并且可能缺乏领域知识。在实践中，当遵循适当程序时，主观编码者通常被使用且可接受。

**内容分析范围**：明确定义什么构成数据集——考虑是否包含公共消息、私人消息或两者。将人群限制在数据所代表的范围内。考虑数据包含的时间段。在上下文中分析数据——不同的上下文需要不同的解释。

**文本与多媒体内容**：文本内容分析遵循既定的编码程序。没有自动化工具，多媒体分析更加劳动密集。尽可能使用部分自动化注释：手动编码子集，训练系统，然后自动注释其余部分。为大型多媒体数据集选择子集进行手动分析。

**信度阈值**：科恩卡帕系数有公认的解释：K ≤ 0.20（差），0.20 < K ≤ 0.40（一般），0.40 < K ≤ 0.60（中等），0.60 < K ≤ 0.80（满意），K > 0.80（近乎完美）。值低于 0.60 表示信度有问题，需要改进。

## 5. 书本案例与示例

**姑息治疗 IT 支持研究**：Kuziemsky 及其同事对七名专业人员（护士、医生、咨询师）的 50 小时访谈、患者图表和研究文献进行了扎根理论分析。该研究使用开放编码和轴心编码形成了对姑息治疗重要因素之间关系的详细图谱。这示范了扎根理论如何处理来自多个来源的数据以构建理论理解。

**唐氏综合症儿童计算机使用研究**：研究人员在调查计算机使用行为时使用了涌现编码和先验编码的混合。第三、四和五级的大多数代码项目是通过涌现编码识别的，而“人类能力”下的类别来自现有的心理学理论。代码结构被构建为具有多层细节的层次格式。“课程整合”等术语直接借用自家长的回应作为实境编码。

**在线社区信任研究示例**：用于说明数据集定义如何影响分析。研究人员研究人际信任发展必须决定是否包含公共公告板消息、私人即时消息交换或两者。选择取决于研究问题（社区氛围影响与私人通信）和实际访问限制。

**儿童计算机使用调查（练习）**：访谈了 50 名 8-15 岁儿童的练习。计算机使用目标被分为三类：教育、沟通和娱乐。两名编码者独立编码数据，一致矩阵显示教育方面 0.49 一致，沟通方面 0.11，娱乐方面 0.23。此示例演示了如何计算科恩卡帕系数并评估编码者间信度。

**姑息治疗开放编码和轴心编码示例**：分析产生了一个编码图，显示“时间”、“空间”和“政策”等概念如何与姑息治疗维度相关。“疼痛的时间发作”和“分诊和评估”等开放代码从原始数据中识别，然后分组为轴心类别。这说明了迭代编码过程如何构建对复杂现象的理解。

**用户负担量表研究**：Suh 及其同事开发了一个包含六个构念的用户负担模型和一个用户负担量表。他们使用效标效度（与既定测量的相关性）和构念效度（因子分析）来衡量模型和量表的有效性。这示范了效度测试在实践中的应用。

**自动化多媒体注释**：部分自动化方法涉及人工编码者手动注释多媒体数据的子集，然后使用手动编码的数据训练应用程序将低级特征与高级概念联系起来。一旦建立了概念检测器，它就可以自动注释其余数据。这应用于图像、视频和音频剪辑。

**老年人计算机困难示例**：用于说明理论框架如何指导编码。当研究老年人使用计算机时遇到的困难时，完善的人类能力框架（认知、身体、感知）提供了高级编码类别。参与者的回应被分组到这三个类别中，示范了先验编码如何使用现有理论。

## 6. 常见错误与陷阱

**将编码与解释混淆**：一个常见的误解是编码只不过是解释文本和计算关键词。编码涉及与数据互动、进行比较、派生概念，并根据这些概念的属性和维度来发展它们。这是一个更深层次的分析过程。

**在开放编码期间感到不知所措**：新手编码者可能觉得难以在数据中识别任何有趣的东西，或者相反，可能觉得数据如此丰富，以至于他们需要编码几乎每一个字。这两种极端都是有问题的。解决方案是寻找特定类型的陈述（目标、行动、结果、后果、原因、背景、策略），对数据提出问题，并进行比较。

**在扎根理论中从先入为主的理论想法开始**：由于扎根理论不是从预先形成的假设或理论开始，研究人员必须从没有先入为主的理论想法开始，以便概念和理论真正从数据中涌现。先入为主的意见可能会偏倚发现。

**未能明确定义数据集**：包含不符合定义标准的数据会污染数据集并产生偏倚或误导性结果。例如，当研究侧重于公共社区互动时分析私人消息，或包含指定时间段之外的数据。

**在数据分析中忽略上下文**：任何词语、术语和主张都必须在其提取的特定上下文中解释。不考虑上下文分析数据就像拿苹果和梨比较。必须在多个层面考虑上下文——既包括数据集的高级上下文，也包括单个短语、句子或段落的低级上下文。

**未使用多名编码者**：使用单一编码者消除了检查信度的可能性。多名编码者应独立编码数据，并应计算信度测量（科恩卡帕系数）。不同的编码视角可以加强分析。

**选择性使用数据**：研究人员可能无意识地强调支持其首选理论的观察，同时淡化与替代解释一致的观察。有效的定性分析需要给予每个数据点适当的关注并考虑替代解释。

**忽视信度检查**：未计算或报告科恩卡帕系数或其他信度测量使得无法评估编码的质量。信度检查应在整个编码过程中频繁进行，而不是仅在最后进行。

**仅使用百分比一致性**：百分比一致性不考虑编码者之间偶然的一致性。即使随机编码，两名编码者也可能在相当大比例的案例上达成一致。科恩卡帕系数通过测量超越偶然性的一致性来纠正这一点。

**未充分培训编码者**：未能制定明确的编码说明并适当培训所有编码者会导致不一致的编码。编码者在开始之前必须完全理解每个编码项目。如果未达到信度目标，则需要重新培训和修订编码簿。

**将扎根理论视为纯粹线性**：扎根理论的描述（开放编码→概念→类别→理论）暗示了一个线性过程，但实际分析是迭代和非线性的。通过开放编码识别新代码可能导致重新审视早期文档，代码之间新的关系可能在任何阶段出现。

**过度依赖自动化多媒体分析**：由于低级特征和高级概念之间的语义差距，完全自动化的注释技术仍然极易出错。虽然部分自动化方法显示出前景，但应谨慎使用完全自动化分析，并应验证结果。

**未能记录证据链**：没有将分析结果链接回原始数据的清晰文档，读者无法验证结论是否得到证据的支持。一个组织良好的数据库，包含所有材料、程序和分析产品，对于建立效度至关重要。

**未考虑对立解释**：仅呈现一种解释而不考虑替代方案会削弱效度。展示模型如何比合理的替代方案更好地拟合数据的讨论比孤立地呈现单一模型更有说服力。

**主观编码者导致夸大的信度**：当制定编码方案的同一个人也进行编码时，由于对编码含义的隐性知识，报告的信度可能被夸大。缓解策略包括使用客观编码者、偏见意识培训和仔细的文档记录。
