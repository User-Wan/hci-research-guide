# 第16章：与残疾研究参与者合作 — 知识点

## 1. 核心概念定义

**残疾 (Disability)** 在 HCI 研究中涵盖影响感觉（视觉、听觉）、运动（行动能力、灵活性）、认知（记忆、注意力、学习）和心理功能的广泛状况。本章强调残疾不是二元的——它存在于一个光谱上，许多人同时经历多种残疾。研究人员必须了解与其研究相关的残疾的具体特征。

**辅助技术 (Assistive Technology)** 指帮助残疾人完成原本可能困难或不可能的任务的设备、软件或系统。示例包括屏幕阅读器（JAWS、NVDA、VoiceOver）、屏幕放大器、替代输入设备（开关、眼动追踪仪、吹吸设备）、助听器、人工耳蜗和通信辅助工具。了解辅助技术对于研究可访问性的 HCI 研究人员至关重要。

**屏幕阅读器 (Screen Reader)** 是为盲人或视力低下用户朗读计算机屏幕内容的软件。流行的屏幕阅读器包括 JAWS（Job Access With Speech）、NVDA（NonVisual Desktop Access）和 VoiceOver（内置于 Apple 设备中）。屏幕阅读器通过可访问性 API 解释网页和应用程序，按照文档结构确定的线性顺序读取内容。

**Web 内容可访问性指南 (Web Content Accessibility Guidelines, WCAG)** 是使 Web 内容对残疾人可访问的国际公认标准。WCAG 2.0/2.1 定义了四个原则：内容必须是可感知的、可操作的、可理解的和健壮的 (POUR)。每个原则都有特定的指南和可在三个符合级别（A、AA 和 AAA）测试的成功标准。

**第 508 条 (Section 508)** 是一项美国联邦法律，要求联邦政府开发、采购、维护或使用的电子和信息技术对残疾人可访问。第 508 条标准引用了 WCAG 2.0 AA 级用于 Web 内容。

**通用设计 (Universal Design)** 是将产品和环境设计为尽可能让所有人都能使用，而无需适应或专门设计。在 HCI 中，通用设计意味着创建对具有不同能力的用户（包括残疾人）都有效的界面。

**屏幕放大器 (Screen Magnifier)** 是为视力低下用户放大屏幕一部分的软件。用户通过在屏幕上移动放大查看窗口来导航。流行的屏幕放大器包括 ZoomText 以及 Windows 和 macOS 中内置的放大功能。

**替代输入设备 (Alternative Input Devices)** 是允许运动残疾用户与计算机交互的硬件设备。示例包括开关（单个或多个）、眼动追踪仪、头部追踪仪、吹吸设备（由呼吸控制）、轨迹球和改良键盘。许多运动残疾用户使用替代输入设备的组合。

**认知负荷 (Cognitive Load)** 指使用界面所需的心理努力。当界面复杂、使用术语、需要记忆先前步骤或同时呈现太多信息时，认知残疾用户可能会经历更高的认知负荷。减少认知负荷使所有用户受益。

**残疾人物画像 (Personas with Disabilities)** 是代表可能使用产品或服务的具有特定残疾的用户原型。创建残疾人物画像有助于设计团队在整个设计过程中考虑可访问性，而不是事后考虑。

**与残疾用户的参与式设计 (Participatory Design with Disabled Users)** 涉及将残疾人作为设计过程中的积极合作伙伴，而不仅仅是研究对象。这种方法认识到残疾人是自己经验的专家，可以提供非残疾研究人员可能错过的见解。

**残疾参与者招募 (Recruitment of Participants with Disabilities)** 带来了独特的挑战，包括较小的可用人群、交通困难、对专业设备的需求以及与残疾人组织和倡导团体合作的重要性。

**残疾参与者的知情同意 (Informed Consent for Participants with Disabilities)** 可能需要额外考虑，包括替代格式（大字、盲文、音频）、为认知残疾参与者简化语言、在适当时涉及护理人员或法定监护人，并确保同意是真正知情和自愿的。

**通用可用性 (Universal Usability)** 是使信息和通信服务对每个人都可用的目标，无论残疾、识字能力、技术访问或其他因素。它将通用设计原则扩展到数字领域。

## 2. 理论框架

**残疾的社会模型 (Social Model of Disability)**：该模型区分了损伤（身体或精神状况）和残疾（阻止有损伤的人充分参与的社会障碍）。在 HCI 中，这意味着当界面以排斥具有某些损伤的用户的方式设计时，界面就造成了残疾。可访问性的责任在于设计师和开发者，而不是用户。

**残疾的医学模型 (Medical Model of Disability)**：这种传统模型将残疾视为个体中需要修复或适应的问题。在 HCI 中，这种模型导致将“特殊”的可访问性功能作为事后添加。社会模型在 HCI 研究中通常更受青睐，因为它侧重于消除障碍而不是“修复”用户。

**可访问性-可用性集成框架 (Accessibility-Usability Integration Framework)**：可访问性和可用性不是单独关注的问题，而是深度交织在一起的。对残疾人可访问的界面通常对每个人都更可用。例如，字幕使聋人受益，但也帮助在嘈杂环境中的用户；键盘导航使运动障碍用户受益，但也帮助高级用户。

**补偿和适应模型 (Compensation and Adaptation Model)**：残疾人开发策略和变通方法来完成任务。了解这些补偿策略可以深入了解他们面临的挑战和潜在的设计解决方案。HCI 研究应研究这些适应以指导更好的设计。

**包容性设计框架 (Inclusive Design Framework)**：设计应从设计过程开始就考虑人类多样性的全部范围，而不是事后考虑。这包括能力、语言、文化、性别、年龄和其他形式的人类差异。包容性设计使每个人受益，而不仅仅是残疾人。

**专业知识和熟悉度考虑 (Expertise and Familiarity Considerations)**：残疾参与者可能对辅助技术具有不同程度的专业知识，这可能显著影响他们与所研究界面的交互。研究人员必须将辅助技术专业知识视为一个变量，就像他们考虑计算机和领域专业知识一样。

## 3. 方法论步骤

### 步骤1：了解具体的残疾

在招募参与者之前，彻底研究与您的研究相关的具体残疾。了解：残疾的性质（感觉、运动、认知或组合）、它如何影响技术使用、常用哪些辅助技术、可能需要哪些适应，以及每个残疾类别内经验的光谱。认识到残疾不是单一的——两个具有相同诊断的人可能有非常不同的经验和能力。

### 步骤2：与残疾人组织合作

与为残疾人服务的组织合作：
- **国家组织**：国家盲人联合会、国家聋人协会、残疾人倡导组织
- **地方组织**：独立生活中心、残疾人服务提供者
- **在线社区**：特定残疾的论坛、社交媒体群组
- **教育机构**：盲人/聋人学校、大学残疾人服务

这些组织可以帮助招募参与者、为研究提供空间、提供关于适应的专业知识并验证研究设计。考虑将组织作为顾问支付费用。

### 步骤3：规划适应

设计研究以适应参与者的需求：
- **物理可访问性**：确保研究地点轮椅可通行，具有适当的照明，并且没有物理障碍
- **设备**：提供适当的辅助技术或验证参与者可以自带
- **材料**：以可访问的格式准备同意书和研究材料（大字、盲文、音频、简化语言）
- **时间**：为可能需要更慢完成任务的参与者允许额外时间
- **交通**：如果交通是障碍，考虑在参与者所在位置（家、社区中心、工作场所）进行研究

### 步骤4：调整知情同意

为残疾参与者修改同意流程：
- 以替代格式提供同意书（大字、盲文、音频、数字可访问格式）
- 使用清晰、简单的语言，特别是对于认知残疾参与者
- 对于无法自己提供同意的参与者，遵循监护人或代理同意的法定要求，同时仍寻求参与者的同意
- 通过适当的沟通方法确保参与者理解研究
- 为同意过程允许额外时间

### 步骤5：调整研究程序

修改研究程序以使其可访问：
- **对于盲人参与者**：确保所有研究材料对屏幕阅读器可访问；提供视觉元素的音频描述；不要仅依赖视觉线索
- **对于聋人/重听参与者**：提供书面说明；考虑手语翻译；使用视觉线索；确保视频内容有字幕
- **对于运动残疾参与者**：允许额外时间；确保界面可以用替代输入设备操作；如果存在替代方案，不要求特定的运动操作（例如使用鼠标）
- **对于认知残疾参与者**：简化说明；将任务分解为更小的步骤；提供提醒；减少干扰

### 步骤6：适当地收集数据

根据参与者的能力调整数据收集：
- 标准可用性指标（任务完成时间、错误率）可能无法在具有不同残疾的参与者之间直接比较
- 考虑除了定量指标外，还收集关于经验的定性数据
- 注意残疾参与者可能使用不同的策略来完成任务
- 不要假设较慢的完成时间表示可用性较差——交互可能只是不同
- 收集关于辅助技术使用作为研究一部分的数据

### 步骤7：在残疾背景下分析结果

在解释结果时考虑：
- 参与者的残疾和辅助技术使用如何影响他们的表现
- 观察到的困难是由于界面设计还是辅助技术的限制
- 结果是否会推广到不同的辅助技术
- 发现如何为通用设计原则提供信息
- 避免基于参与者的诊断假设他们能做什么或不能做什么

### 步骤8：负责任地报告发现

在报告发现时：
- 详细描述参与者的残疾和辅助技术使用
- 讨论研究期间所做的适应
- 在上下文中呈现结果——不要简单地比较残疾和非残疾参与者
- 强调设计影响而不是参与者的“缺陷”
- 遵循任何发布的报告或论文的可访问性指南

## 4. 决策标准

**何时包含残疾参与者**：在以下情况下包含残疾参与者：
- 界面旨在供一般使用（应对所有人可访问）
- 界面专门为残疾用户设计
- 可访问性是研究或设计目标
- 法律要求强制可访问性（例如第 508 条）
- 研究旨在理解通用设计原则

**选择研究设计**：考虑是否：
- **在一般研究中包含残疾参与者**：测试通用可用性；可能需要适应
- **进行残疾特定研究**：允许集中调查；可能产生更详细的发现
- **比较残疾和非残疾用户**：揭示差异可用性；将残疾框架为缺陷的风险
- **使用参与式设计**：让残疾用户作为设计合作伙伴；最具包容性的方法

**招募策略**：对于残疾参与者：
- 与残疾人组织合作（付费咨询安排）
- 提供在参与者首选地点进行研究
- 提供交通援助或补偿
- 允许灵活安排
- 提供更高的补偿以反映额外的时间和精力
- 与残疾人社区建立长期关系

**适应级别**：根据以下因素确定需要哪些适应：
- 参与者的具体残疾
- 研究任务的要求
- 可用资源和设备
- 法律要求（ADA、第 508 条）
- 最少限制环境原则

**辅助技术考虑**：决定是否：
- 使用参与者自己的辅助技术（最自然，但在参与者之间有所不同）
- 提供标准化的辅助技术（参与者之间一致，但可能不熟悉）
- 研究界面与特定辅助技术之间的交互
- 用多种辅助技术进行测试以确保广泛兼容性

**样本量考虑**：涉及残疾参与者的研究通常面临较小的可用人群。对招募可能性要现实。具有 2-3 名参与者的案例研究对于深入理解可能很有价值。受控实验可能需要接受较小的样本并承认局限性。考虑多站点研究以增加样本量。

## 5. 书本案例与示例

**盲人用户的菜单任务绩效研究**：研究人员研究了视力正常用户的宽而浅与窄而深菜单结构发现是否适用于盲人屏幕阅读器用户。他们与国家盲人联合会 (NFB) 合作，后者帮助确定参与者并提供研究空间。NFB 作为顾问获得报酬，参与者获得了高于惯例的补偿。研究发现，盲人用户与视力正常的用户一样，在宽而浅的菜单结构中表现更好——验证了某些可用性原则在不同用户群体中的推广性。

**Web 内容可访问性测试**：自动化工具（Deque WorldSpace、WAVE、A-Checker）根据 WCAG 2.0 指南检查网页，识别缺少替代文本、颜色对比度不足和缺少表单标签等问题。然而，自动化工具无法评估替代文本是否有意义、内容是否逻辑组织或用户体验是否令人满意。与辅助技术用户进行手动测试对于全面的可访问性评估至关重要。

**屏幕阅读器可用性研究**：比较不同屏幕阅读器（JAWS、NVDA、VoiceOver）的研究发现，性能在不同屏幕阅读器和不同网站之间存在显著差异。熟悉一个屏幕阅读器的用户可能在另一个屏幕阅读器上遇到困难，即使在同一网站上执行相同的任务。这凸显了使用参与者实际使用的辅助技术进行测试的重要性。

**移动界面的可访问性**：研究检查了视力障碍者如何使用带有屏幕阅读器（iOS 上的 VoiceOver、Android 上的 TalkBack）的触摸屏智能手机。这些研究揭示了独特的挑战，包括缺乏触觉反馈、需要复杂手势以及导航未考虑可访问性的应用程序的困难。

**认知可访问性研究**：认知残疾用户的研究检查了简化语言、减少认知负荷、提供清晰导航和使用一致布局如何提高可用性。这些研究通常使用参与式设计方法，让认知残疾用户作为积极的设计合作伙伴，而不是被动的研究对象。

**辅助技术与界面交互**：研究检查了不同辅助技术如何与 Web 内容交互。例如，屏幕阅读器可能以不同方式解释 ARIA（可访问的富互联网应用程序）属性，动态内容更新可能不会向屏幕阅读器用户宣布，除非正确编码。这些研究为辅助技术开发和 Web 可访问性标准提供了信息。

**通用设计案例研究**：HCI 中通用设计的示例包括：视频字幕（使聋人用户和在嘈杂环境中的用户受益）、语音控制（使运动障碍用户和手忙碌的用户受益）、高对比度模式（使视力低下用户和在明亮阳光下的用户受益）以及键盘快捷键（使运动障碍用户和高级用户受益）。

**与残疾用户的参与式设计**：在整个开发过程中将残疾人作为设计合作伙伴的项目产生更可访问和可用的界面。这些项目认识到残疾用户是自己经验的专家，可以提供非残疾设计师可能错过的见解。参与式设计对于开发辅助技术特别有价值。

## 6. 常见错误与陷阱

**将残疾视为单一的**：假设所有具有特定残疾的人都是一样的。两个盲人用户可能使用不同的屏幕阅读器，具有不同的专业知识水平，并具有不同的完成任务策略。两个轮椅使用者可能具有不同的手臂功能水平。始终了解参与者的具体特征和能力。

**使用非残疾代理人**：要求非残疾参与者模拟残疾（例如，戴眼罩模拟失明）会产生误导性结果。残疾人已经开发出补偿策略和辅助技术专业知识，而非残疾人缺乏这些。模拟残疾研究错过了这些重要因素。

**忽视辅助技术**：未能考虑辅助技术如何与所研究的界面交互。网站在技术上可能可访问，但由于结构不良、缺少标签或混乱的导航，仍然难以用屏幕阅读器使用。始终使用参与者实际使用的辅助技术进行测试。

**将残疾框架为缺陷**：以残疾参与者“不能做什么”而不是界面如何未能可访问的方式呈现问题。问题在于设计，而不是用户。将发现框架为设计建议而不是参与者的局限性。

**适应不足**：未能提供必要的适应（可访问格式、额外时间、适当设备、物理可访问性）排斥参与者并违反伦理原则。仔细规划适应并询问参与者需要什么。

**未与残疾人组织合作**：试图在没有为残疾人服务的组织帮助下招募残疾参与者。这些组织拥有个别研究人员缺乏的既定关系、专业知识和访问权限。建立真正、尊重的伙伴关系。

**补偿不足**：提供的补偿没有考虑到残疾参与者可能需要投入的额外时间和精力。交通挑战、较长的任务完成时间以及对专业设备的需求都增加了残疾参与者参与的成本。

**假设残疾群体内的同质性**：将所有盲人用户、所有聋人用户或所有轮椅使用者视为相同。残疾经历因以下因素而有很大差异：发病年龄、严重程度、辅助技术专业知识、教育和个人应对策略。

**未使研究材料可访问**：同意书、问卷、任务说明和其他研究材料对残疾参与者不可访问，将他们排除在参与之外。从一开始就以可访问的格式准备材料。

**在不可访问的地点进行研究**：选择轮椅不可通行、照明不良或缺乏适当适应的研究地点。将物理环境作为研究设计的一部分考虑。

**忽视认知可访问性**：仅关注感觉和运动可访问性而忽视认知可访问性。认知残疾用户面临独特的挑战，包括：复杂语言、记忆需求、注意力要求和信息过载。设计考虑认知可访问性的研究。

**未报告可访问性细节**：未能在研究报告中描述参与者的残疾、辅助技术使用和适应。没有这些信息，读者无法正确解释结果或评估推广性。始终包含残疾背景的详细描述。

**将可访问性视为检查清单**：将可访问性作为合规练习（满足 WCAG 标准）而不是可用性目标。技术合规性不能保证良好的用户体验。始终包括与残疾人的用户测试以验证界面是否真正可用。

**从小样本过度概括**：从仅包含少数残疾参与者的研究中得出广泛的结论。虽然由于招募挑战，小样本研究通常是必要的，但研究人员应承认其样本的局限性并避免过度概括。将发现框架为见解而不是确定性结论。
